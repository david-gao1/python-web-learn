# 每周多篇博客发布模板（每篇 <= 5000 字）

## 核心规则

- 每周可发布 1-3 篇，按知识点拆分。
- 每一篇都必须同时包含“概念讲解 + 实战示例”。
- 禁止发布纯概念文章或纯实战文章。
- 每篇字数上限 5000 字。

## 按知识点拆分示例（第 1 周）
- 篇 1：`__len__` + `__iter__`（容器行为）
- 篇 2：`__contains__` + 查询设计（成员判断）
- 篇 3：`__repr__` + 调试可观测性（可读输出）

## 单篇固定结构（强制）
1. 要解决什么问题
2. 概念部分：核心思路与关键机制
3. 实战部分：代码片段 + 运行命令 + 结果说明
4. 工程验收：lint/类型检查/测试结果
5. 思考题（2-3 道）

## 字数控制建议
- 开场场景：300-500 字
- 概念部分：800-1500 字
- 实战部分：800-1500 字
- 工程验收：300-600 字
- 思考题与结尾：200-400 字
- 合计建议：2500-4500 字

## 发布前检查清单
- 本文是否 <= 5000 字
- 本文是否同时包含“概念 + 实战”
- 是否包含可运行命令
- 是否包含关键代码片段
- 是否说明了“为什么这样设计”
- 是否有 2-3 道思考题
