# Python 教程计划（对齐《流畅的 Python（第2版）》目录）

## 教程定位
- 学习顺序以书本目录为主线：第一部分到第五部分，覆盖第 1-24 章。
- 产出导向：每个阶段必须交付可运行代码 + 自动化测试 + 博客文章（按知识点拆分）。
- 工具链统一：uv 管理环境，ruff/mypy/pytest 做质量门禁。

## 固定规则（本仓库约束）
- 默认按知识点拆分为 1-3 篇教学文章，每篇建议 <= 5000 字，且必须同时包含：问题、概念、机制、实战、思考题。
- 允许额外提供“合并长文”（按章/按阶段汇总），字数可 > 5000，用于完整叙事与复盘。
- 每个阶段的最小门禁：
  - `uv run ruff check .`
  - `uv run mypy .`
  - `uv run pytest -q`

## 文档与解释规则
- 当引入新的语言特性（装饰器、描述符、数据类、类型重载、协议方法等）时：
  - 在代码中补齐 docstring（作用、触发方式、工程取舍）。
  - 在文章中补齐直觉 + 机制 + 用法 + 常见误区。
- 统一参考文档：`docs/python_decorators_and_annotations_vs_java.md`

## 推荐节奏（14 周，按章节密度拆分）

### 第 1 周（第 1 章）：Python 数据模型
- 交付代码：一个“像内置容器一样工作”的领域对象（当前选择 `OrderBook`）
- 核心能力：`__len__`、`__iter__`、`__contains__`、`__repr__`、`__getitem__`
- 博客拆分（示例）：
  - `__len__ + __iter__`
  - `__contains__`
  - `__repr__`
  - `__getitem__`

### 第 2 周（第 2 章）：丰富的序列
- 交付代码：一个面向真实数据的“序列处理管道”（不依赖数据库）
- 关键知识点：推导式/生成器表达式、拆包、切片、序列模式匹配、增量赋值陷阱
- 博客拆分建议：
  - “拆包 + 切片：把数据清洗写短”
  - “序列模式匹配：把分支变成结构化规则”
  - “`+=` 与可变序列：工程里最常见的坑”

### 第 3 周（第 3 章）：字典和集合
- 交付代码：一个“事件去重 + 聚合”的小型索引服务（纯内存）
- 关键知识点：dict 的视图/推导、set 的运算、`defaultdict`/`Counter`/`ChainMap`
- 博客拆分建议：
  - “用 dict 表达业务索引：键设计与边界”
  - “用 set 做幂等：集合运算替代 if/else”

### 第 4 周（第 4 章）：Unicode 文本和字节序列
- 交付代码：一个可复现的“文本/字节处理工具”（读取文件、编码转换、摘要输出）
- 关键知识点：编码/解码边界、`bytes`/`bytearray`、文本规范化、二进制协议入门
- 博客拆分建议：
  - “编码错误从哪来：边界在哪里就在哪里处理”
  - “`bytes` 不是字符串：二进制数据的工程约束”

### 第 5 周（第 5 章）：数据类构建器
- 交付代码：把第 1-3 周的小对象升级为“可比较、可序列化、可验证”的数据模型
- 关键知识点：`dataclasses`、`namedtuple`、`__slots__`、不可变对象、构造与校验
- 博客拆分建议：
  - “数据类不是语法糖：它在维护什么不变量”
  - “`__slots__` 与内存：什么时候值得”

### 第 6 周（第 6 章）：对象引用、可变性和垃圾回收
- 交付代码：一组可复现的“可变性陷阱”实验用例 + 修复策略
- 关键知识点：别名、浅拷贝/深拷贝、函数默认参数陷阱、引用循环与 GC
- 博客拆分建议：
  - “默认参数的坑：为什么它不是每次都重新创建”
  - “拷贝策略：什么时候 shallow 足够”

### 第 7 周（第 7 章）：函数是一等对象
- 交付代码：一个可插拔的“策略函数注册表”（为第 10 章铺垫）
- 关键知识点：可调用对象、函数签名、`operator`、高阶函数与组合
- 博客拆分建议：
  - “把业务规则当函数：从 if/else 到策略表”

### 第 8 周（第 8 章）：函数中的类型提示
- 交付代码：把第 7 周策略系统加上类型边界（输入/输出契约）
- 关键知识点：`Callable`、`TypeVar`、重载、`TypedDict`（与后续呼应）
- 博客拆分建议：
  - “类型提示不是注释：它如何减少跨团队误用”

### 第 9 周（第 9 章）：装饰器和闭包
- 交付代码：对策略系统加“缓存/重试/限流”装饰器（可测试）
- 关键知识点：闭包、装饰器顺序、参数化装饰器、`functools` 工具
- 博客拆分建议：
  - “装饰器的工程用途：把横切逻辑变成可组合模块”

### 第 10 周（第 10 章）：使用一等函数实现设计模式
- 交付代码：把第 7-9 周能力组织成一个最小可用规则引擎
- 关键知识点：策略/命令/模板方法的函数化表达，减少样板与继承
- 博客拆分建议：
  - “用一等函数替代继承：哪些模式会消失”

### 第 11 周（第 11-12 章）：Pythonic 对象 + 序列的特殊方法
- 交付代码：升级 `OrderBook`（或另一个领域对象）为“完整序列体验”
- 关键知识点：`__eq__`/`__hash__`/`__bytes__`/格式化、序列协议全家桶
- 博客拆分建议：
  - “对象协议：让你的类接入更多内建能力”
  - “序列特殊方法：从能迭代到像 list 一样好用”

### 第 12 周（第 13-14 章）：协议/ABC + 继承取舍
- 交付代码：定义 `Protocol` 接口 + 两个实现（组合优先）
- 关键知识点：`Protocol` vs ABC、MRO、混入、组合与继承的边界
- 博客拆分建议：
  - “接口先行：用 Protocol 限定协作边界”
  - “继承的成本：什么时候应该退回组合”

### 第 13 周（第 15-16 章）：类型提示进阶 + 运算符重载
- 交付代码：一个泛型容器/工具类 + 最小运算符重载示例（可读、可测）
- 关键知识点：重载签名、运行时类型提示、型变、数值/向量类运算符设计
- 博客拆分建议：
  - “重载签名与 TypedDict：把 API 约束写进类型系统”
  - “运算符重载：何时提升可读性，何时制造困惑”

### 第 14 周（第 17-24 章）：控制流与元编程（分段推进）
- 交付代码：一个“可观测的数据处理服务”作为收官项目（同步 + 并发/异步 + 类型边界）
- 第 17 章：迭代器/生成器/协程（流式处理）
- 第 18 章：with/match/else（资源安全 + 结构化控制流）
