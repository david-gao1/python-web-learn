# AGENTS

本文件定义本仓库的协作与交付规范，用于指导人类与自动化 Agent 在仓库中持续迭代内容。

## 目标
- 学习路径严格对齐《流畅的 Python（第2版）》目录。
- 每章（或每阶段）必须产出：可运行代码、自动化测试、教学文章。
- 以工程可复现为第一优先级：命令可运行、结果可验证、质量门禁可通过。

## 工具链与门禁
- 使用 `uv` 管理虚拟环境与依赖（仓库根目录维护统一 `.venv`）。
- 默认质量门禁：
  - `uv run ruff check .`
  - `uv run mypy .`
  - `uv run pytest -q`

## 目录结构约定
- `chapters/`：按章组织的学习内容；每章至少包含 `src/`、`tests/`、`README.md` 与文章文件。
- `docs/`：仓库级规范与模板。
- `flasky/`：第三方学习项目，不参与本仓库门禁（lint/typecheck/test）。

## 教学文章规范
- 每一篇文章必须包含：问题、概念、机制、实战、思考题。
- 默认按知识点拆分为 1-3 篇，每篇建议 <= 5000 字。
- 允许额外提供“合并长文”（按章/按阶段汇总），字数可 > 5000，用于完整叙事。

## 代码内文档规范
- 当引入新的语言特性（例如装饰器、描述符、数据类、类型重载、协议方法）时：
  - 在代码中用 docstring 解释其作用与工程取舍（优先写在类/方法 docstring 中）。
  - 同步在文章中补充“直觉 + 机制 + 用法 + 边界”。
- 仓库中关于“Python 装饰器/类型注解 vs Java 注解”的统一说明文档：
  - `docs/python_decorators_and_annotations_vs_java.md`
