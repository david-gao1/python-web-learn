# Python 教程计划（基于《流畅的 Python（第2版）》）

## 目标
- 面向已有基础语法的工程师，构建可用于真实项目的 Python 工程能力。
- 以“理解 + 应用”为主线，重点覆盖数据模型、类型系统、并发和异步。
- 每周产出可运行代码与可验证结果，而不是只写概念笔记。

## 学习节奏
- 周期：8 周
- 每周结构：问题场景 -> 核心机制 -> 关键代码 -> 实战任务 -> 验收
- 每周交付：`src/` 代码 + `tests/` 测试 + 1 份简短复盘 + 博客文章（可多篇，每篇 5000 字以内，且每篇都含概念 + 实战）

## 8 周安排

### 第 1 周：Python 数据模型（第 1 章）
- 主题：特殊方法与 Pythonic API（`__len__`、`__iter__`、`__repr__`）
- 实战：实现一个可迭代、可统计、可调试输出的领域对象
- 产出：`OrderBook` 或 `TaskQueue` 领域模型

### 第 2 周：序列、映射与模式匹配（第 2-3 章）
- 主题：拆包、切片、字典模式、`match` 的工程用法
- 实战：事件解析器（把 JSON 事件映射为内部命令）
- 产出：可扩展的解析模块，新增事件无需改主流程

### 第 3 周：函数是一等对象与装饰器（第 7-10 章）
- 主题：策略函数、闭包、装饰器、函数式组合
- 实战：可插拔规则引擎（策略注册、重试、缓存）
- 产出：规则引擎最小可用版本与回归测试

### 第 4 周：协议与类型系统（第 11-15 章）
- 主题：`Protocol`、`TypedDict`、泛型与类型约束
- 实战：统一存储接口（本地存储 + 云存储两种实现）
- 产出：`mypy --strict` 通过的接口驱动代码

### 第 5 周：迭代器与生成器管道（第 17 章）
- 主题：惰性计算、流式处理、`yield from`
- 实战：日志 ETL（读取 -> 过滤 -> 聚合 -> 输出）
- 产出：百万行输入下内存稳定的处理管道

### 第 6 周：上下文管理与资源安全（第 18 章）
- 主题：`with`、上下文管理器、异常路径资源释放
- 实战：数据库会话/文件句柄安全封装
- 产出：异常路径可验证的资源管理组件

### 第 7 周：并发模型与执行器（第 19-20 章）
- 主题：线程、进程、执行器选型与性能边界
- 实战：并发下载 + 多进程 CPU 校验
- 产出：串行/线程/进程三组 benchmark 报告

### 第 8 周：asyncio 工程实践（第 21 章）
- 主题：任务编排、限流、超时、取消、错误收敛
- 实战：异步抓取服务（限速 + 重试）
- 产出：可压测的异步服务与集成测试

## 每周统一验收（Definition of Done）
- 功能可运行：有明确输入和输出，示例可复现。
- 质量可检查：`ruff check .`、`mypy .`、`pytest -q` 通过。
- 结果可度量：至少 1 项指标（时延、吞吐、内存、错误率）。
- 设计可解释：提交简短说明，回答“为什么这样设计”。
- 博客可发布：每周按知识点拆分为 1-3 篇教程博客，每篇不超过 5000 字，且每篇必须同时包含概念讲解与实战示例。

## 最终项目（第 8 周收官）
- 目标：实现一个“小型可观测数据处理服务”。
- 要求：同步 + 异步两版核心链路，包含类型约束与测试。
- 验收：完整运行文档、质量门禁通过、关键指标达标。
